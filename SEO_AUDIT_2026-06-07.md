Scope: headless Next.js / WooCommerce SEO audit only, no code changes.

- Canonical/query-parameter logic review

## Crawl/index audit results
- Sampled URLs: 48 total.
  - `/shop`, `/categories`, `/brands`, `/concerns`, `/ingredients`, `/skin-type`, `/new-arrivals`, `/sale`
  - 20 product pages
  - 5 category pages
  - 5 brand pages
  - filter/query parameter URLs
  - fake 404 tests
- All sampled core pages returned `200` except one brand fake URL that correctly returned `404`.
- Product pages have clean absolute canonical URLs and meta descriptions.
- Product JSON-LD and breadcrumb schema are present in raw HTML for sampled product pages.
- `/concerns?concern=acne-blemish-care` correctly redirects to the clean path `/concerns/acne-blemish-care`.

## Query parameter / canonical issues found
- Good canonical handling:
  - `/shop?sort=rating` → canonical `/shop`
  - `/shop?utm_source=google` → canonical `/shop`
  - `/shop?srsltid=1234` → canonical `/shop`
- Problematic cases:
  - `/shop?filter=sale` keeps canonical `https://e-mart.com.bd/shop?filter=sale` instead of clean `/shop`.
  - `/shop?price_min=500&price_max=1500` keeps canonical with query params and HTML-escaped ampersand.
- This suggests an inconsistent filter/price query canonicalization risk.

## Soft 404 / error handling issues
- `https://e-mart.com.bd/category/not-a-real-category-test-123`
  - returns `200`
  - title `Category Not Found | Emart`
  - robots `index, follow`
  - Conclusion: soft 404 issue.
- `https://e-mart.com.bd/shop/not-a-real-product-test-123`
  - returns `200`
  - title `Page Not Found | Emart`
  - robots `noindex, nofollow`
  - Conclusion: better but still not a true `404`.
- `https://e-mart.com.bd/brands/not-a-real-brand-test-123`
  - returns `404` correctly.

## Product discovery / pagination
- `/shop` raw HTML contains product links.
- Category and list pages are crawlable via anchor links in sample page HTML.
- No audit evidence was collected that product discovery is only infinite-scroll-only.

## Key findings
1. Product pages and core catalog pages are generally healthy.
2. Sitemap is valid and clean in sampled review.
3. Live product structured data is present and looks correct.
4. Filter and price query URLs are inconsistently canonicalized.
5. Invalid category URLs return soft 404s and should instead return proper `404`/`notFound()`.
6. Invalid product path returns `200` with `noindex`, which is a weaker pattern than a true `404`.

## Recommended next steps
- Fix invalid category slugs to return true `404` responses.
- Normalize shop filter and price query URL canonicalization to the clean `/shop` page or the correct canonical base.
- Reinspect robots handling for `/*?concern=`, `/*?skin_type=`, and `/*?ingredient=` after canonical/query cleanup.
- After fixes, retest sitemap, sample pages, and GSC URL inspection for the fixed pages.
