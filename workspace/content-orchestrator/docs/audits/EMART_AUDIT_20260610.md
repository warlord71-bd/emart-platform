# E-Mart Platform Audit — 2026-06-10

Auditor: Claude Code (read-only session). Scope: Next.js frontend (`/var/www/emart-platform/apps/web` — note: prompt referenced `/var/www/emartweb`, which does not exist; PM2 process `emartweb` runs from the former), WordPress/WooCommerce backend via code reading + live GET requests to `https://e-mart.com.bd`. Expo app excluded. ~43 live GET requests used.

## Executive Summary

**Overall health grade: B+.** The platform is in strong technical shape: schema/canonical/sitemap discipline is excellent, checkout is server-validated with idempotency, security headers are comprehensive, and crawl hygiene (real 404s, clean redirects, AI-crawler-friendly robots.txt + llms.txt) is above average for ecommerce. Top 5 issues:
1. **Admin auth returns the shared `REVALIDATE_SECRET` as its session token** — secret reuse across admin orders + cache revalidation, no rate limit (C-01).
2. **PDPs serve `cache-control: private` and `force-dynamic`** — zero CDN caching on the highest-traffic page type (H-01).
3. **Homepage product rails are `ssr:false`** — no product links in homepage server HTML; crawlers see 2 `/shop/` links (H-02).
4. **Product schema availability diverges from purchasability logic** (backorder/managed-stock edge cases) (H-03).
5. **aggregateRating absent from all Product schema** — no star snippets in SERPs despite a reviews system existing (H-04).

## Critical

- **C-01 — Admin token is the revalidate secret, unthrottled.** [src/app/api/admin/auth/route.ts:31](apps/web/src/app/api/admin/auth/route.ts#L31) returns `process.env.REVALIDATE_SECRET` as the admin token after a plain `!==` credential compare; [src/app/api/admin/orders/route.ts:19-20](apps/web/src/app/api/admin/orders/route.ts#L19) also accepts it via `?token=` query param (leaks into access logs/referrers). No rate limiting on the auth endpoint enables credential brute force; a leaked token grants both customer-order data and cache-purge power. *Why it matters:* one secret compromise exposes customer PII and admin capability. *Fix:* issue a separate short-lived signed token for admin sessions, drop `?token=` support, add rate limiting and constant-time comparison.

*No other Critical findings. No hardcoded `ck_`/`cs_`/API keys found anywhere in source; Woo credentials are server-side only.*

## High

- **H-01 — PDP/category pages are `force-dynamic` with `private` cache.** [src/app/shop/[slug]/page.tsx:388-389](apps/web/src/app/shop/[slug]/page.tsx#L388) sets `dynamic = 'force-dynamic'` AND `revalidate = 3600` (force-dynamic wins; the revalidate line is dead). Live PDP responds `cache-control: private, max-age=300, must-revalidate`, `cf-cache-status: EXPIRED`; one sampled PDP took 3.4s total vs ~70ms for cached pages. Same pattern at [src/app/category/[slug]/page.tsx:383-385](apps/web/src/app/category/[slug]/page.tsx#L383) (category currently rescued by the Nginx `s-maxage` override; PDPs are not). *Why:* every PDP hit is a full SSR render, no CDN shielding — slow for users and origin-load-heavy for ~3,600 products. *Fix:* decide intentionally — either ISR (`revalidate` only, drop `force-dynamic`) or keep dynamic but emit `s-maxage` headers for PDPs as done for categories.
- **H-02 — Homepage product links invisible to crawlers.** [src/components/home/HomepageDeferredSections.tsx:24-40](apps/web/src/components/home/HomepageDeferredSections.tsx#L24) loads FlashSale/ProductGrid sections with `dynamic(..., { ssr: false })` behind an intersection observer. Live homepage HTML contains only 2 `/shop/` hrefs out of 218 links. *Why:* homepage link equity to products is lost; Google's renderer may never trigger the observer. *Fix:* server-render at least the first product rail (links + names) and defer only the interactive enhancement — note the 2026-06-05 perf gains and freeze rules; needs owner approval to change homepage rendering.
- **H-03 — Schema availability diverges from real purchasability.** [src/lib/seo/product.ts:87-89](apps/web/src/lib/seo/product.ts#L87) maps `stock_status !== 'instock'` → `OutOfStock`, but [src/lib/stock.ts:74-88](apps/web/src/lib/stock.ts#L74) treats `onbackorder` as available and rejects managed-stock items with insufficient quantity even when `stock_status === 'instock'`. *Why:* GMC/rich-result mismatch risk — schema can claim InStock for items checkout rejects, and OutOfStock for purchasable backorder items. *Fix:* derive schema availability from `normalizeStockAvailability()` and map backorder to `schema.org/BackOrder`.
- **H-04 — aggregateRating absent from all Product schema.** Verified on 3 live PDPs (COSRX essence, Eucerin cream, Kwailnara lotion): `aggregateRating` and `review` are absent while a reviews system exists (`getProductReviews`, [src/lib/woocommerce.ts:1405](apps/web/src/lib/woocommerce.ts#L1405), and `/api/product-reviews`). *Why:* no star snippets in search results — direct CTR loss against competitors. *Fix:* include `aggregateRating` + top `review` entries in Product JSON-LD when `rating_count > 0`.
- **H-05 — No rate limiting on any public API route.** Checked `/api/checkout`, `/api/search`, `/api/newsletter/subscribe`, `/api/admin/auth`, `/api/track-order` — zero throttling anywhere. *Why:* COD order spam is a known Bangladesh ecommerce attack; checkout creates real Woo orders + customers per request. *Fix:* add per-IP rate limiting at Nginx or middleware for checkout/auth/newsletter at minimum.
- **H-06 — `wp-login.php` publicly reachable (HTTP 200) on the production domain.** Live check: `curl -sI https://e-mart.com.bd/wp-login.php` → 200, while `/wp-json/` is correctly 403-blocked. *Why:* exposes the WordPress backend (contradicting the headless-privacy policy) and provides a brute-force surface. *Fix:* IP-allowlist or Cloudflare Access on `wp-login.php`/`wp-admin` (owner decision — do not break legitimate admin access).

## Medium

- **M-01 — MPN fabricated from internal SKU; no GTIN.** Product schema emits `"mpn": "EM-93028"` — the internal `EM-` SKU, not a manufacturer part number; `gtin` absent (live PDP JSON-LD). *Why:* incorrect MPN can trigger GMC disapprovals/mismatches. *Fix:* drop `mpn` unless a real manufacturer number exists; add GTIN/barcode where available.
- **M-02 — GA4 ecommerce funnel incomplete.** Only `purchase` is sent to GA4 ([src/app/order-success/OrderSuccessClient.tsx:81](apps/web/src/app/order-success/OrderSuccessClient.tsx#L81)); no `view_item`, `add_to_cart`, `begin_checkout` (those exist for Meta/Reddit only). *Why:* GA4 funnel reports and Google Ads optimization signals are blind above purchase. *Fix:* mirror the Meta event call sites with `trackGA4` ecommerce events.
- **M-03 — All pixels deferred 30s or until interaction.** [src/app/runtime-widgets.tsx:18-40](apps/web/src/app/runtime-widgets.tsx#L18) gates GA4/Meta/Reddit behind `useDeferredThirdParty(30000)`. *Why:* visitors who bounce in <30s without touching the page are never measured — bounce rate and paid-traffic attribution undercount (intentional perf tradeoff; record it as accepted or shorten the timer). *Fix:* confirm tradeoff with owner; consider 5–10s timer or server-side page_view via CAPI-style proxy.
- **M-04 — Root layout canonical = homepage inherited by non-overriding pages.** [src/app/layout.tsx:67-68](apps/web/src/app/layout.tsx#L67) sets `alternates.canonical: SITE_URL`; the 404 page emits `canonical: https://e-mart.com.bd` (verified live on a bogus category URL — noindex is present, limiting damage). *Why:* any future page that forgets to set its own canonical silently canonicals to the homepage. *Fix:* remove the canonical from root layout metadata; set it explicitly per page (all sampled real pages already do).
- **M-05 — Four parallel price-formatting implementations.** `formatPrice` in [src/lib/woocommerce.ts:1545](apps/web/src/lib/woocommerce.ts#L1545), `formatBDT` in [src/lib/formatters.ts:10](apps/web/src/lib/formatters.ts#L10), plus local copies in [src/app/skin-quiz/SkinQuizClient.tsx:157](apps/web/src/app/skin-quiz/SkinQuizClient.tsx#L157) and [src/components/layout/Header.tsx:83](apps/web/src/components/layout/Header.tsx#L83). *Why:* BDT formatting drift risk across surfaces. *Fix:* consolidate on `formatBDT` and delete the rest.
- **M-06 — Design tokens bypassed by raw hex.** 33× `#9f1239` and 26× `#D4A248` hardcoded in TSX despite `accent`/`brass` tokens existing in [tailwind.config.js:23-40](apps/web/tailwind.config.js#L23); 190 arbitrary Tailwind values (`p-[…]`-style) in components; legacy `#1a1a2e` survives in [src/styles/tokens.css:6](apps/web/src/styles/tokens.css#L6) (`--color-brand-dark`) and [src/components/social/VideoCard.tsx:67](apps/web/src/components/social/VideoCard.tsx#L67). *Why:* palette changes won't propagate; old-brand color still referenced. *Fix:* sweep hex → token classes; retire `--color-brand-dark` or re-point it to the Porcelain ink color.
- **M-07 — Three theme systems coexist.** `porcelain`-era Tailwind tokens, `lumiere-*` utility classes, and `data-theme="midnight-blossom"` CSS-var scoping ([src/app/categories/page.tsx:89](apps/web/src/app/categories/page.tsx#L89), [src/components/home/ShopByCategory.tsx:28](apps/web/src/components/home/ShopByCategory.tsx#L28), 20 files total). *Why:* style drift and unclear source of truth per surface. *Fix:* document which theme owns which surface, plan consolidation post-freeze.
- **M-08 — `src/lib/woocommerce.ts` is a 1,558-line god-module with 29 `any`s.** 72 `any` usages app-wide concentrate here; [src/components/layout/Header.tsx](apps/web/src/components/layout/Header.tsx) is 1,188 lines mixing search, mega-menu, cart, account. *Why:* highest-churn files are the least type-safe and hardest to review. *Fix:* split woocommerce.ts by domain (products/categories/orders/reviews) and type the API responses.
- **M-09 — Several lib fetchers have no timeout.** `src/lib/wordpress-posts.ts`, `sitemapEntries.ts`, `youtubeRss.ts`, `seo.ts` use bare `fetch()` with try/catch but no `AbortSignal.timeout` (the axios Woo client correctly uses 8s). *Why:* a hung upstream can stall page renders/sitemap generation indefinitely. *Fix:* add `signal: AbortSignal.timeout(8000)` to each.
- **M-10 — Checkout error passthrough leaks backend messages.** [src/app/api/checkout/route.ts:205](apps/web/src/app/api/checkout/route.ts#L205) returns `error?.response?.data?.message` from Woo/plugin directly to the client. *Why:* internal error details (plugin names, validation internals) reach the browser. *Fix:* map known error codes to friendly messages; log the rest server-side only.

## Low

- **L-01 — Dead atomic-design scaffolding.** `src/components/{atoms,molecules,organisms,templates}/` contain only index.ts files (64 lines total) referenced by nothing. *Fix:* move to attic.
- **L-02 — `categories/page.tsx` uses raw `JSON.stringify` for JSON-LD.** [src/app/categories/page.tsx:92](apps/web/src/app/categories/page.tsx#L92) bypasses the `safeJsonLd` helper used elsewhere (`</script>` injection only if category data is hostile — low likelihood). *Fix:* use `safeJsonLd` for consistency.
- **L-03 — Hardcoded analytics ID fallbacks.** `G-WMJNX87Q2N` at [src/app/layout.tsx:17](apps/web/src/app/layout.tsx#L17) and Meta Pixel `763041131179021` at [src/components/analytics/MetaPixel.tsx:9](apps/web/src/components/analytics/MetaPixel.tsx#L9) as `||` fallbacks; `NEXT_PUBLIC_GOOGLE_TAG_ID` is absent from `.env.local` so the hardcoded value is what actually runs. *Why:* env gating is illusory; staging builds would fire production analytics. *Fix:* require env vars, render nothing when unset.
- **L-04 — Stale `updatedDate` literals.** [src/lib/best-definitions.ts:36,130,214](apps/web/src/lib/best-definitions.ts#L36) hardcode `2026-05-19`; these will age while content claims freshness. *Fix:* tie to content edits or remove.
- **L-05 — Search suggestion images have empty alt.** `/api/search` returns Woo image `alt: ""` for some products (verified live, CeraVe query). *Why:* minor a11y gap in the suggestion dropdown. *Fix:* fall back to product name (PDP gallery already does this at [src/components/product/ProductImage.tsx:39](apps/web/src/components/product/ProductImage.tsx#L39)).
- **L-06 — `.env.local.backup-20260502-google-restore` lingers in app root.** Contains live-shaped secrets duplicated outside the canonical env file. *Fix:* move to `/root/.attic-*` (user-approved move only — flagged, not done).
- **L-07 — `priceValidUntil` uses UTC date.** [src/lib/seo/product.ts:33-37](apps/web/src/lib/seo/product.ts#L33) — rolling +1 year, correct in design (no fixed-date time bomb found); the UTC slice can lag Dhaka local date by up to 6h. Cosmetic.

## Verified-Good (do not re-audit)

- **Schema suite:** WebSite+OnlineStore / Organization / LocalBusiness / Product / BreadcrumbList / FAQPage / CollectionPage+ItemList all present and parse-clean on home, category, 3 PDPs, CMS page; offers include `hasMerchantReturnPolicy` and `shippingDetails`; prices/currency (BDT) correct; one `h1` per page; OG + twitter cards complete; self-referencing absolute canonicals on all real pages.
- **Crawl hygiene:** robots.txt correct with GPTBot/ClaudeBot/PerplexityBot/Google-Extended allowed; llms.txt live (7.5KB); sitemap has 3,627 product URLs with 3,278 distinct (real) lastmod values; 4 random sitemap URLs → all 200; bogus URL → real 404; http→https and www→apex → 301; trailing slash → 308 canonical.
- **Checkout API:** server-side subtotal + shipping quote recomputation, live stock verification per line item, idempotency key, 50-item/99-qty caps, email validation, Meta CAPI purchase with event-id dedup, guest→customer password-reset flow. Order creation via secret-protected mu-plugin endpoint.
- **Security posture:** full header set live (HSTS w/ preload, X-Frame-Options DENY, nosniff, Referrer-Policy, Permissions-Policy, strict CSP with frame-ancestors 'none'); `/wp-json/` 403; `/.env` 404; no secrets in `NEXT_PUBLIC_*`; no Woo keys in client components; product/blog HTML sanitized (`sanitizeHtml`) before `dangerouslySetInnerHTML`; `/api/version` exposes only BUILD_ID.
- **Search:** `/api/search` validates/clamps input, noindex+follow with canonical on /search, debounced + AbortController in header, handles misspellings via Woo partial match, <1s responses.
- **Performance basics:** TTFB ~50ms on cached pages (measured from same datacenter — not representative of BD users); all 5 fonts via `next/font/google`; no moment/lodash; next/image with srcset/quality everywhere sampled; footer year dynamic; homepage `x-nextjs-cache: HIT` with `s-maxage=3600`.
- **Marketing-collection exclusion:** centralized in `src/lib/product-utils.ts`, consumed by `seo/product.ts` and PDP; `/api/search` independently skips redirect-category slugs — no bypassing code path found.
- **Stock model:** `src/lib/stock.ts` is the single shared authority for UI + checkout (schema being the one exception, see H-03).
