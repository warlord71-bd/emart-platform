# Task Board First Rule

Owner preference: before starting any non-trivial work, check `workspace/TASKS.md` and update or add the relevant task first. Do not jump directly into implementation just because the next action seems obvious.

Apply this especially to SEO, CRO, AI assistant, content, deployment, data, and storefront changes. The task board should show what is planned, in progress, or blocked before code/catalog/system changes begin.
